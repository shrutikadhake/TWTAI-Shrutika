The Four Skill Patterns

PatternPurposeKey Question
ReviewerCheck content against criteria"Does this meet the standard?"
GeneratorCreate new content from source"Make this exist."
TransformerConvert form or style"Keep substance, change expression."
AnalyzerExtract insights from content"What's in here? What's missing?"

Pattern 1: The Reviewer

Purpose: Check content against criteria and report findings.

You are a [ROLE] reviewing [TARGET] for [PURPOSE].
Review: $ARGUMENTS

## Checklist
1. [Criterion] — [what PASS looks like]
2. [Criterion] — [what PASS looks like]

## Output
| Criterion | Status | Finding | Action |
Status: ✅ PASS | ⚠️ NEEDS WORK | ❌ MISSING

## Summary
Score: X/Y | Top 3 improvements by impact

Examples: /check-api-doc, /check-readme, /term-audit

Pattern 2: The Generator

Purpose: Create new content from source material.

Read the source material at $ARGUMENTS and generate [OUTPUT TYPE].

## Rules
- [What must be included]
- [What must be excluded]
- [Formatting rule]

## Output
Output ONLY the [content type].
No preamble. No commentary. Begin with the first line of content.

Examples: /release-notes, /scaffold-api-doc, /build-glossary

7 / 13

Pattern 3: The Transformer

Purpose: Convert content from one format or style to another.

Transform the content at $ARGUMENTS.

## From: [current format/style/level]
## To:   [target format/style/level]

## Preserve (do NOT change)
- Code samples, proper nouns, product names

## Change
- [What must be transformed]
- [Style rules to apply]

## Output
The transformed content only. No markup showing changes.

Examples: /active-voice, /to-reference, /simplify

8 / 13

Pattern 4: The Analyzer

Purpose: Extract insights, patterns, or gaps from raw material.

Analyze the content at $ARGUMENTS for [ANALYSIS PURPOSE].

## What to Analyze
- [Dimension 1]
- [Dimension 2]

## Output
[SECTION 1: Summary Statistics]
[SECTION 2: Detailed Findings]
[SECTION 3: Priority Recommendations — each starts with a verb]

Examples: /doc-coverage, /jargon-audit, /naming-audit

Which Pattern Do I Use?

What is your primary goal?
│
├── Evaluate existing content ─────────→ REVIEWER
│     "Does this meet the standard?"
│
├── Create new content ─────────────→ GENERATOR
│     "Make this exist."
│
├── Change the form of content ─────→ TRANSFORMER
│     "Same substance, different form."
│
└── Extract information ────────────→ ANALYZER
      "Tell me what's in here / missing."

Workshop: Build One of Each

Work individually or in pairs.

Assignments (choose or assign):

  • /check-style → Reviewer pattern
  • /release-notes → Generator pattern
  • /simplify-prose → Transformer pattern
  • /doc-coverage → Analyzer pattern

For each skill:

  1. Pick the right pattern template
  2. Fill in YOUR Role, Context, Checklist/Rules, Constraints, Output
  3. Test on a sample document
  4. Note one thing you'd improve
